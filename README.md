# Ionic TS Auth Tabs
Приложение демонстрирует экран логина и основное приложение c тремя вкладками: Главная, Заказы, Профиль. Авторизация условная (локальная) — если оба поля (username и password) заполнены, пользователь попадает в приложение.

## Основные требования, учтённые при разработке
### Экран авторизации:
- Наполнение: поля: username, password, кнопка Войти.
- Если оба поля не пустые → переход в основное приложение (с табами).
- Если хотя бы одно пустое → отображается ошибка: Введите логин и пароль.

### Вкладки:
- Главная — приветственный текст Добро пожаловать!.
- Заказы — список фейковых заказов (IonCard).
- Профиль — отображает username и кнопку Выйти (очищает состояние).

### Управление состоянием: 
- React Context (AuthContext).

### Минимум одна кастомная компонента: 
- LoginForm (инкапсулирует поля, валидацию и кнопку).

### Поддержка навигации через react-router (встроенный в Ionic).

## Технологический стек:
- React
- TypeScript
- Ionic Framework (React)
- Vite
- React Router
- React Context
